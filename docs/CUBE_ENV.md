# Cube (PushCube) 环境分析

本文档基于当前仓库中的 `CubeEnv` 单方块版本（`env_type='single'`）整理，重点面向 counterfactual dynamics evaluation。这里的 “PushCube” 对应当前代码中的单方块 Cube 任务。

## 1. 初始化逻辑

### 1.1 默认起始位置与朝向分布

单方块任务集包含 5 个预定义任务模板。每个模板都给定一个初始中心位置和一个目标中心位置，单位均为米：

- 任务 1：初始 `(0.425, 0.1, 0.02)`，目标 `(0.425, -0.1, 0.02)`
- 任务 2：初始 `(0.35, 0.0, 0.02)`，目标 `(0.50, 0.0, 0.02)`
- 任务 3：初始 `(0.50, 0.0, 0.02)`，目标 `(0.35, 0.0, 0.02)`
- 任务 4：初始 `(0.35, -0.2, 0.02)`，目标 `(0.50, 0.2, 0.02)`
- 任务 5：初始 `(0.35, 0.2, 0.02)`，目标 `(0.50, -0.2, 0.02)`

在 `task` 模式下：

- 每个 episode 会先随机采样一个任务模板。
- Cube 初始 `x,y` 不是全空间均匀采样，而是在模板初始点附近加入独立均匀扰动，范围为 `[-0.01, 0.01]` 米。
- Cube 初始 `z` 固定为 `0.02` 米。
- Cube 初始 yaw 每个 episode 独立均匀采样于 `U(0, 2π)`。
- 对应姿态四元数是“绕 z 轴旋转 yaw”得到的单位四元数。

如果只看 variation space 的默认初值，而不是实际 reset 之后的训练分布：

- `start_position` 默认值是 `(0.3, 0.0)`
- `start_yaw` 默认值是 `0`

但在 `task` 模式下，这两个默认值不会直接定义训练初始状态，实际起点仍由“任务模板 + 1cm 扰动”和“随机 yaw”决定。

### 1.2 Target 默认分布

`task` 模式下，目标位置不是连续均匀分布，而是离散地来自上面的 5 个任务模板目标点：

- `(0.425, -0.1, 0.02)`
- `(0.50, 0.0, 0.02)`
- `(0.35, 0.0, 0.02)`
- `(0.50, 0.2, 0.02)`
- `(0.50, -0.2, 0.02)`

同时：

- 目标 `z` 固定为 `0.02` 米。
- 目标朝向固定为单位四元数，也就是不要求任何目标旋转。

`data_collection` 模式下则不同：

- 非堆叠目标位置来自连续均匀采样。
- 采样边界为 `x ∈ [0.3, 0.55]`，`y ∈ [-0.3, 0.3]`。
- 目标 yaw 独立均匀采样于 `U(0, 2π)`。

因此，若讨论“训练/评估主分布”，应优先理解为 `task` 模式下的离散目标分布，而不是 `data_collection` 模式的连续目标分布。

### 1.3 训练集中的 Cube 空间边界

对于单方块 `task` 模式，真实训练起点来自 5 个模板初始点附近各自 `±1cm` 的扰动并集，因此整体可达范围为：

- `x` 约在 `[0.34, 0.51]` 米
- `y` 约在 `[-0.21, 0.21]` 米
- `z = 0.02` 米

需要区分两类“边界”：

- 训练分布边界：上面的 `[0.34, 0.51] × [-0.21, 0.21]`
- 环境允许的 object sampling bounds：`x ∈ [0.3, 0.55]`，`y ∈ [-0.3, 0.3]`

前者更适合用于 counterfactual dynamics evaluation 中的 in-distribution 分析；后者更适合判断 near-OOD 或 data collection 范围。

## 2. 成功判定

### 2.1 成功测量方式

成功判定非常直接：

- 取 Cube 当前 3D 中心位置
- 取目标 3D 位置
- 计算两者欧氏距离
- 若距离 `<= 0.04` 米，则该 Cube 成功

对于单方块任务：

- 只有一个 Cube
- 因此整体任务成功等价于该 Cube 成功

这意味着成功判定本质上不是“Cube 的 xy 中心是否落入某个目标框”，而是“Cube 中心是否进入一个以目标点为中心、半径 4cm 的 3D 球形邻域”。由于单方块目标与方块通常都在桌面高度 `z=0.02`，实际效果接近平面距离阈值判定。

### 2.2 阈值与是否考虑旋转

具体阈值为：

- 位置误差阈值：`0.04` 米，即 `4 cm`

旋转方面：

- 不检查 Cube 朝向是否与目标对齐
- 不检查 yaw 误差
- 即使目标 yaw 被设置，也不会进入 success check

因此，这个环境的成功标准只依赖位置，不依赖姿态。

### 2.3 最大步数

`CubeEnv` 本体没有写死一个内部 time-limit 截断逻辑，最大步数来自外层 world 或 wrapper 配置，而不是环境类本身。

在当前仓库的常见使用方式中：

- OGB 数据采集配置常用 `max_episode_steps = 200`
- 规划评估配置中，常令 `max_episode_steps = 2 * eval_budget`
- 例如 `eval_budget = 50` 时，常设置为 `100`

另外：

- 当 `terminate_at_goal=True` 时，一旦成功，episode 会直接 `terminated=True`
- 当 `terminate_at_goal=False` 时，成功后也不会立即终止，可继续 rollout

因此，若要报告“最大步数”，应明确说明它是外部配置项，不是 `CubeEnv` 内部固定常数。

## 3. 因果因素与随机化

### 3.1 固定变量

默认 reset 逻辑下，以下因素是固定的或默认不随 episode 变化：

- Cube 物体类别固定为立方体
- 单方块任务中 Cube 数量固定为 1
- 成功阈值固定为 `4 cm`
- 桌面/地面布局固定
- 机械臂结构、夹爪结构、控制方式固定
- 物理时间步与控制时间步固定
- 工作空间边界固定
- `task` 模式下目标朝向固定为单位四元数

此外，虽然 variation space 中定义了 Cube 颜色、Cube 尺寸、agent 颜色、相机角度、光照强度等变量，但默认 reset 并不会自动对所有这些变量进行采样。默认启用的 variation 主要只有位置与 yaw。

从当前代码可见，以下物理属性也没有在 episode 级别显式随机化：

- Cube 形状
- Cube 质量
- Cube 摩擦
- 桌面摩擦
- 桌面材质与几何结构

因此，如果做 counterfactual dynamics evaluation，这些物理因素更像“外部干预变量”，而不是训练时原生随机化变量。

### 3.2 每回合随机化的变量

在 `task` 模式下：

- 任务 ID：从 5 个单方块模板中随机采样
- Cube 初始 `x,y`：在模板初始点附近做独立均匀扰动 `[-0.01, 0.01]` 米
- Cube 初始 yaw：均匀采样 `U(0, 2π)`

在 `data_collection` 模式下：

- Cube 初始位置：从 object/start position 边界内采样
- Cube 初始 yaw：均匀采样 `U(0, 2π)`
- 目标位置：有一部分概率采样为 stacking 目标，否则从 target bounds 内均匀采样
- 目标 yaw：均匀采样 `U(0, 2π)`
- 目标块索引：从可作为目标的顶部方块集合中随机选取

对单方块版本来说：

- `target_block` 实际上总是唯一那个 Cube
- stacking 分支在单方块环境下没有实际意义

### 3.3 随机化方法

当前环境中的随机化方法主要有三种：

- 离散均匀采样：从预定义任务模板中随机选一个任务
- 连续均匀采样：位置扰动、目标位置、yaw 角等
- 条件分支采样：例如 data collection 中先决定是否采用 stacking 目标，再决定目标位置

更具体地说：

- `task` 模式的初始位置分布是“离散模板中心 + 小范围连续均匀扰动”
- `task` 模式的目标位置分布是“离散模板目标点”
- `data_collection` 模式的初始位置与目标位置分布则是“连续矩形区域均匀采样”

## 4. 对 Counterfactual Dynamics Evaluation 的启示

对当前单方块 Cube / PushCube 环境而言，最关键的结论是：

- 训练分布中的位置变化范围相对窄，主要围绕 5 个任务模板展开
- 初始朝向分布很宽，yaw 覆盖整个 `[0, 2π]`
- 成功判定只依赖位置，不依赖姿态
- 目标在主训练模式下是离散的，不是连续区域

因此，以下几类干预具有不同含义：

- 改变质量、摩擦、接触参数：主要影响动力学轨迹和最终能否进入 `4cm` 成功半径
- 改变初始 yaw：会显著影响接触动力学，但不会直接进入 success function
- 将目标位置从 5 个离散目标点外推出去：更偏向分布外泛化，而不只是动力学反事实
- 将初始位置从模板附近扩展到完整 object bounds：属于 near-OOD 到 OOD 的分布移动

如果目标是评估“动力学反事实”而非“任务语义反事实”，更建议优先干预：

- Cube 质量
- Cube/桌面摩擦
- 接触恢复系数或阻尼
- 机械臂动作噪声

而不是优先改动目标分布本身，因为后者会把“动力学变化”和“任务分布变化”混在一起。
