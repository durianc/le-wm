## 3. 事实评测（Factual Evaluation）：已知世界观下的预测准确性

### 3.1 事实评测的核心目标与设置

**定义**：事实评测(Factual Evaluation) 评估模型在**真实数据分布**上的预测能力，即给定真实的初始观测、真实的动作序列和真实的环境参数，模型能否准确预测后续的观测或状态。

**数据来源**：
评测时使用的是 `init_data` 生成的验证数据流（`val_loader`）以及由 `eval.yaml` / `data_config.yaml` 组合出的环境配置，而不是报告里所说的一个单独硬编码“离线数据集”。每个 episode 通常记录：
- 初始观测 $o_0$
- 动作序列 $\{a_t\}_{t=0}^{H-1}$
- 相应的真实状态轨迹 $\{s_t\}_{t=0}^{H}$ 或观测序列 $\{o_t\}_{t=0}^{H}$
- 环境参数 $(w_x, w_y)$

**关键特点**：
- 训练/验证数据是否固定墙与门，取决于 `data_config.yaml` 和加载时的 override，不是评测脚本里写死的常量
- 评测侧会复用验证集中的动作与位置序列，再结合环境创建器重放或规划
- `eval.yaml` 里没有把墙/门参数固定为 32/24；这组数值只是在一些任务或可视化脚本里出现

### 3.2 事实评测的执行流程（逻辑概览）

评测流程遵循以下逻辑框架：

```
FOR EACH episode IN validation_dataset:
    1. 初始化：取 episode 中的初始观测 o_{0}
    
    2. 编码：z_{0}^{pred} = Enc(o_{0})
       （注：上标 pred 表示"模型预测"）
    
    3. 多步演化：FOR t = 0 TO H-1:
        z_{t+1}^{pred} = Pred(z_{t}^{pred}, a_{t})
    
    4. 与真实值对比：
       z_{t}^{true} = Enc(o_{t})  [从数据集读取真实观测再编码]
       
    5. 计算误差：
       Loss_{t} = ||z_{t}^{pred} - z_{t}^{true}||²
       
    6. 逐步积累：记录每一步的误差

计算统计指标：
    - 各时刻的平均 MSE：mean_mse_{t} = E[Loss_{t}]
    - 误差的标准差：std_mse_{t} = Std[Loss_{t}]
    - 位置空间的误差：通过"位置探针"从 z 预测 (x,y) 后计算欧氏距离
```

源码中的实际调用顺序是：`main.py` 在 `eval_only_mode` 下先执行 `launch_unroll_eval`，再执行 `launch_plan_eval`，最后才是 `launch_counterfactual_eval`；在训练循环中则是按计划评测、轻量 unroll 评测、反事实评测的顺序按频率触发，而不是单独一个统一的评测入口。

### 3.3 事实评测的物理解释

从世界模型的视角，事实评测衡量的是：

**预测模型在已知真实初值与动作驱动下的轨迹跟踪能力**。

具体地：
- 如果 MSE 随时刻增长缓慢（如线性或亚线性增长），说明模型习得了稳定的近似动力学
- 如果 MSE 快速增长（如指数增长），说明模型存在**混沌敏感性**或动力学学习不足
- 如果误差在早期时刻就很大，说明编码器难以从观测中提取充分统计量

**因果性角度**：
在真实数据生成的制程下，因果图为：
$$a_t \rightarrow s_{t+1}, \quad w_x, w_y \rightarrow s_{t+1} \text{ (via constraints)}$$

事实评测隐含地假设：**动作和初值充分解释了后续状态的变化**。
